# PowerPoint Status Report Structure

## Presentation Overview

**Purpose:** Executive-level project status communication  
**Audience:** Stakeholders, sponsors, leadership  
**Frequency:** Weekly or bi-weekly  
**Duration:** 5-10 minute read/presentation  

## Slide Deck Structure (7 Slides)

### Slide 1: Title & RAG Status
**Layout:** Title slide with status indicator

**Elements:**
1. **Title Section (Top 40%):**
   - Project Name (Large, Bold, 36pt)
   - Reporting Period (e.g., "Week of Jan 15-19, 2024")

2. **RAG Status Badge (Center):**
   - Large circular or rectangular badge
   - Color: Red (Critical), Amber/Yellow (At Risk), Green (On Track)
   - Text: "PROJECT STATUS: GREEN"

## Design Guidelines

### Typography
- **Headers:** Calibri Bold or Arial Bold, 28-36pt
- **Body Text:** Calibri or Arial, 16-20pt
- **Footnotes:** 10-12pt
- **Line Spacing:** 1.2-1.5 for readability

### Color Palette
**Primary:**
- Title Background: Company brand color
- Accent: Complementary brand color

**Status Indicators:**
- Success/Green: #70AD47
- Warning/Amber: #FFC000
- Critical/Red: #FF0000
- Neutral/Gray: #7F7F7F

**Backgrounds:**
- Slide Background: White or light gray (#F5F5F5)
- Section Headers: Dark blue (#203864) or brand color

### Layout Consistency
- Margins: 0.5 inch on all sides
- Logo: Top right corner, consistent across slides
- Slide numbers: Bottom right
- Footer: Project name + date (bottom left)

### Visual Hierarchy
1. Most important: Top and center
2. Supporting info: Middle sections
3. Details/disclaimers: Bottom and small font

## Chart Best Practices

### Budget Charts
- **Avoid:** 3D effects, too many colors
- **Use:** Flat design, 2-3 colors maximum
- **Label:** Clear axis labels, data labels on bars

### Timeline Charts
- **Avoid:** Overcrowding milestones
- **Use:** 5-7 key milestones only
- **Highlight:** Current status with vertical line

### Trend Charts
- **Avoid:** More than 3 lines on one chart
- **Use:** Line charts for time-series data
- **Annotate:** Key events affecting trends

## Presentation Tips

### For Live Presentation (10 min)
- **Slide 1:** 30 seconds (set context)
- **Slide 2:** 2 minutes (executive summary)
- **Slide 3-4:** 2 minutes (accomplishments + next week)
- **Slide 5:** 3 minutes (issues deep-dive)
- **Slide 6:** 1.5 minutes (metrics)
- **Slide 7:** 1 minute (actions/decisions)

### For Email Distribution
- Add speaker notes with detailed explanations
- Ensure all acronyms defined
- Include contact info for follow-up
- Attach supporting documents (risk register, Gantt)

### For Executive Audience
- Lead with Slide 2 (summary) verbally
- Only dive into Slide 5 (issues) if red/amber status
- Be ready with mitigation plans
- Know your budget and schedule variance numbers cold

## Automation Opportunities

### Data Sources
Pull from:
- Project management tools (Jira, Asana burndown)
- Financial systems (budget actuals)
- Version control (commits, PR stats)
- Survey tools (team morale)

### PowerPoint Automation
Use Python `python-pptx` library:
```python
from pptx import Presentation

# Load template
prs = Presentation('status-report-template.pptx')

# Update text placeholders
slide = prs.slides[1]
for shape in slide.shapes:
    if shape.has_text_frame:
        if "{{PROJECT_NAME}}" in shape.text:
            shape.text = "My Project Name"

# Update charts
chart = slide.shapes[5].chart
chart.data.categories = ['Week 1', 'Week 2', 'Week 3']
chart.data.values = [10, 15, 20]

prs.save('status-report-week5.pptx')
```

## Template Variations by Methodology

### Agile/Scrum Variant
- Replace "This Week/Next Week" with "Sprint N Summary"
- Add slide: "Velocity & Burndown"
- Add: "Story Points Completed"
- Include: "Definition of Done Met"

### Waterfall Variant
- Emphasize: Phase gate status
- Add: "Deliverables Completed"
- Include: "Earned Value Analysis"
- Show: "Baseline vs Actual Variance"

### Hybrid Variant
- Separate sections for predictive phases vs adaptive sprints
- Dual metrics: EV for infra + velocity for features
- Highlight integration points
