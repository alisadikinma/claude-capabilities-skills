## PACING RULES (CRITICAL)

| Timing | Rule | Why |
|--------|------|-----|
| 0-3s | **Change visual every 1-2s** | Pattern interrupt, stop scroll |
| 3-15s | **Show, don't tell** | Demonstration > explanation |
| 15-18s | **Visual proof only** | Numbers speak louder than words |
| 18-30s | **One CTA, no rambling** | Confusion kills conversion |

## PRODUCTION CHECKLIST

**Pre-Production:**
- [ ] Problem is SPECIFIC (not "debugging is hard" but "finding WHERE bug happens")
- [ ] Solution is ONE thing (not 3 steps)
- [ ] Result is MEASURABLE (not "better" but "30 seconds vs 3 hours")
- [ ] CTA promises the result, not generic

**During Shoot:**
- [ ] Camera stable (tripod or stabilizer)
- [ ] Lighting consistent (no shadows on face)
- [ ] Audio clean (lav mic or boom, no echo)
- [ ] Screen recordings 60fps minimum

**Post-Production:**
- [ ] Cut dead air ruthlessly (pauses <0.5s)
- [ ] Text overlays pop (scale in, not fade)
- [ ] Color grade for pop (increase saturation 10-15%)
- [ ] Sound effects on key moments:
  - Error sound on problem
  - Typing sounds on solution
  - Chime on result
  - Whoosh on CTA

**Quality Check:**
- [ ] Read aloud <30s? (If not, cut words)
- [ ] First 3s grabs attention? (Test with friends)
- [ ] Can understand without audio? (Captions critical)
- [ ] CTA is specific action? (Not "like and subscribe")
