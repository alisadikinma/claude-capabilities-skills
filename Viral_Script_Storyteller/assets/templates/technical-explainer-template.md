### [20-40s] TECHNICAL (How It Actually Works)
**Visual:** Diagram OR code snippet OR system architecture

```
ğŸ“ Script Formula:

Examples:
```

## PRODUCTION GUIDE

### Filming Stages

**Stage 1: Hook (0-5s)**
- Camera: Straight on, medium shot
- Lighting: Bright, face well-lit
- Background: Clean, minimal distraction
- Expression: Curious/inviting

**Stage 2: Analogy (5-20s)**  
- Camera: Can cut to B-roll of analogy
- Voiceover: Record separately, high quality
- Visuals: Stock footage OR simple animation
- Pacing: Relaxed, conversational

**Stage 3: Technical (20-40s)**
- Camera: Screen recording OR diagram animation
- Voiceover: Slightly slower pace (comprehension)
- Visuals: Clean diagrams, high contrast
- Annotations: Highlight key terms as mentioned

**Stage 4: Why It Matters (40-55s)**
- Camera: Case study visuals OR metrics
- Voiceover: Energy picks up (excitement)
- Visuals: Impressive numbers, logos, real examples
- Music: Build toward resolution

**Stage 5: CTA (55-60s)**
- Camera: Back to face, medium shot
- Expression: Friendly, inviting
- Visuals: Minimal text overlay
- Ending: Hold frame 2s for CTA

## PRODUCTION CHECKLIST

**Pre-Production:**
- [ ] Analogy tested with non-technical person (they understand?)
- [ ] Technical explanation <30 words (concise?)
- [ ] Visuals planned for each section (B-roll ready?)
- [ ] One concept only (not mixing 3 ideas?)

**Filming:**
- [ ] Audio quality checked (clear voice, no echo)
- [ ] Screen recordings 1080p+ (readable text?)
- [ ] B-roll gathered (stock or custom shots)
- [ ] Face segments well-lit (no shadows)

**Editing:**
- [ ] Transitions smooth (not jarring cuts)
- [ ] Text overlays readable (font size 36+)
- [ ] Diagrams animated (not static slides)
- [ ] Pacing varied (not monotone throughout)
- [ ] Music supports (doesn't overpower voice)

**Quality Check:**
- [ ] 12-year-old can understand analogy?
- [ ] Technical part still accurate?
- [ ] CTA specific to video topic?
- [ ] Total runtime 30-60s when read aloud?

## PLATFORM OPTIMIZATION

### TikTok
**Hook:** Quick question (2-3s only)  
**Analogy:** Super simple (restaurant, not rocket science)  
**Technical:** Brief (15-20s max)  
**CTA:** "Part 2?" (engagement bait)

### YouTube Shorts  
**Hook:** Can be 5s with intro  
**Analogy:** More detailed okay (20-25s)

**CTA:** "Subscribe for more explainers"

**Hook:** Visual-first (show confusion)  
**Analogy:** Lifestyle connection (coffee shop, not warehouse)  
**Technical:** Diagram-heavy (less code)  
**CTA:** "Save for interview prep"

**Hook:** Professional angle ("This came up in interview")  
**Analogy:** Business context (supply chain, not pizza delivery)

## ADVANCED TECHNIQUES

### Multi-Layer Analogies
For very complex topics, chain analogies:
```
"APIs are like restaurants (basic).
But REST APIs specifically are like a menu with pictures.
GraphQL APIs are like telling the waiter exactly what you want, custom order."
```

### Visual Metaphor Progression
```
Frame 1: Show restaurant exterior (API concept)
Frame 2: Zoom into kitchen (backend)
Frame 3: Show waiter (API endpoint)
Frame 4: Show customer (client app)
Frame 5: Morph entire scene into diagram
```

### Contrast Technique
Show wrong way first:
```
"People think [technical thing] is [complicated misconception].
Actually, it's more like [simple truth]."
```

---

**Remember:** Best explainers make viewer think "Oh, THAT'S what it is!" Not "Hmm, I still don't quite get it." Test your analogy on someone non-technical BEFORE filming.
